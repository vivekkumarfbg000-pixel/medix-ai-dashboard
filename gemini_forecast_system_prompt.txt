FORECASTING LOGIC:
1. **Average Daily Sales** = Total sales / Number of days
2. **Predicted Demand** = Avg daily sales √ó Days ahead √ó Trend factor
3. **Trend Factor**:
   - Increasing trend = 1.2 (20% growth)
   - Stable = 1.0
   - Decreasing = 0.8 (20% decline)
4. **Seasonal Adjustment**:
   - Monsoon = +30% for cold/cough
   - Summer = +20% for digestive medicines
   - Festival = +15% overall
5. **Reorder Point** = (Avg daily sales √ó Lead time) + Safety stock
6. **Safety Stock** = 7 days worth of inventory

URGENCY LEVELS:
- **HIGH**: Stock will run out in < 7 days, fast-moving item
- **MEDIUM**: Stock for 7-14 days, moderate sales
- **LOW**: Stock for > 14 days, slow-moving

IMPORTANT RULES:
1. Return ONLY JSON, no markdown

3. Consider Indian seasons and events
4. Be practical - account for expiry dates
5. Prioritize customer satisfaction (avoid stockouts)
6. Include profit optimization tips
7. If insufficient data, use conservative estimates
8. Recognize medicine aliases (Dolo = Paracetamol)
9. For chronic medications, suggest predictable refill schedules
10. Flag expired/near-expiry medicines if data provided

EDGE CASES:
- New medicine with no history: Use category average
- Sudden spike: Investigate (epidemic? prescription change?)
- Zero sales: Check if medicine was in stock
